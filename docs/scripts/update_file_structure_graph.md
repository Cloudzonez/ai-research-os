# File Structure Graph Generator

## Overview

This document describes the Python script that generates the `FILE_STRUCTURE_GRAPH.md` file in the `docs/` directory.

## Purpose

Automatically generates a visual tree structure of the documentation directory with annotations for key files.

## Original Script Location

Previously: `docs/scripts/update_file_structure_graph.py`  
Now: Documented here for reference

## How It Works

### 1. Configuration

**Root Directory**: `docs/` (parent of scripts directory)  
**Output File**: `docs/FILE_STRUCTURE_GRAPH.md`

**Excluded Directories**:
- `.git`
- `.codex`
- `.agents`
- `__pycache__`

**Excluded Files**:
- `FILE_STRUCTURE_GRAPH.md` (to avoid recursion)

### 2. File Annotations

The script includes descriptions for key documentation files:

| File | Description |
|------|-------------|
| `ARCHITECTURE.md` | High-level architecture notes for the AI-native research platform |
| `BUILD_PLAN.md` | Implementation context and phased build plan |
| `deliverables/export_deck_pdf.py` | Exports the PPT-sized HTML deck to PDF with Playwright |
| `deliverables/research_ai_foundry_deck.html` | PPT-sized HTML presentation deck for the next-generation方案 |
| `deliverables/research_ai_foundry_deck.pdf` | Exported 16:9 PDF version of the presentation deck |
| `deliverables/research_ai_foundry_plan.docx` | Formal方案书 DOCX generated from the Markdown source |
| `deliverables/research_ai_foundry_plan.md` | Formal方案书 source Markdown |
| `deliverables/assets/research-ai-foundry-hero.png` | Generated visual asset used in the DOCX and presentation deck |
| `NEW_RESEARCH_AI_PLAN.md` | New Chinese方案 for an AI-native research operating system |
| `NEXT_GEN_RESEARCH_AI_SYSTEM_IMPROVEMENTS.md` | Upgrade proposal for harness engineering, generated apps, scripts, MCP, and research capability foundry |
| `人工智能赋能教育教学高质量发展20260320.pdf` | Reference PDF reviewed for existing Sichuan Agricultural University AI education plan |
| `西华师范大学AI赋能教育教学进度方案.docx` | Reference docx for AI education application investigation schedule |
| `scripts/update_file_structure_graph.py` | Regenerates this structure graph |

### 3. Tree Generation Algorithm

1. **Recursively traverse** the docs directory
2. **Filter** excluded directories and files
3. **Sort** entries (directories first, then files, alphabetically)
4. **Render** tree structure with proper connectors:
   - `|-- ` for intermediate items
   - `` `-- `` for last items
   - `|   ` for continuation lines
   - `    ` for final branch spacing
5. **Append** annotations from the NOTES dictionary
6. **Write** to `docs/FILE_STRUCTURE_GRAPH.md`

### 4. Output Format

```markdown
# File Structure Graph

Generated by `python3 scripts/update_file_structure_graph.py`.

\`\`\`text
docs/
|-- deliverables/
|   |-- assets/
|   |   `-- research-ai-foundry-hero.png - Generated visual asset...
|   |-- export_deck_pdf.py - Exports the PPT-sized HTML deck...
|   ...
\`\`\`
```

## Usage

### Running the Script

```bash
# From project root
python3 docs/scripts/update_file_structure_graph.py

# Or from docs directory
cd docs
python3 scripts/update_file_structure_graph.py
```

### When to Run

Run this script whenever:
- New documentation files are added
- Documentation structure changes
- File descriptions need updating

### Updating Annotations

To add or modify file descriptions, edit the `NOTES` dictionary in the script:

```python
NOTES = {
    "path/to/file.md": "Description of the file",
    # Add more entries as needed
}
```

## Implementation Details

### Key Functions

**`include(path: Path) -> bool`**
- Filters out excluded directories and files
- Returns True if path should be included in the tree

**`sort_key(path: Path) -> tuple[int, str]`**
- Sorts directories before files
- Alphabetically sorts within each category

**`render_tree(path: Path, prefix: str) -> list[str]`**
- Recursively generates tree structure
- Adds proper connectors and annotations
- Returns list of formatted lines

**`main() -> None`**
- Orchestrates the generation process
- Writes output to FILE_STRUCTURE_GRAPH.md

## Dependencies

- Python 3.6+ (uses f-strings and type hints)
- Standard library only:
  - `pathlib.Path` for file system operations

## Maintenance

This script is self-documenting and requires minimal maintenance. Update the `NOTES` dictionary when adding significant new documentation files.

## Alternative: Manual Updates

If the Python script is not available, you can manually update `docs/FILE_STRUCTURE_GRAPH.md` using:

**Windows PowerShell**:
```powershell
tree /F /A docs > docs/FILE_STRUCTURE_GRAPH.md
```

**Linux/Mac**:
```bash
tree -F docs > docs/FILE_STRUCTURE_GRAPH.md
```

Then manually add the markdown formatting and annotations.
